# Flask项目性能优化说明

## 概述

本项目已完成**三个阶段**的全面性能优化，在保持所有现有功能完全不变的前提下，实现了系统性能的质的飞跃和用户体验的全面提升。

- **爬虫速度提升300%+**：并发处理和网络优化

## 使用说明

### 环境配置

1. **复制环境变量文件**
```bash
cp .env_example .env
```

2. **编辑.env文件，填入实际配置**
```bash
# 必须配置的项目
SECRET_KEY=your-secret-key-change-this-in-production
SILICONFLOW_API_KEY=your_api_key_here
AI_ASSISTANT_PASSWORD=your_password

# 可选的性能配置
CACHE_ENABLED=true
CACHE_TTL=3600
MAX_CACHE_SIZE=1000
BATCH_SIZE=100
MEMORY_LIMIT_MB=512
```

### 性能监控

#### 1. 查看性能统计
访问：`http://your-domain/api/performance/stats`

返回系统性能指标，包括：
- CPU使用率
- 内存使用情况
- 磁盘使用情况
- 缓存统计
- 性能优化建议

#### 2. 查看缓存状态
访问：`http://your-domain/api/cache/stats`

返回缓存统计信息：
- 缓存大小
- 命中率
- 过期时间设置

#### 3. 清空缓存
POST请求：`http://your-domain/api/cache/clear`

手动清空所有缓存数据。

#### 4. 异步任务管理（第二阶段新增）

**获取所有任务状态**
访问：`http://your-domain/api/tasks`

**获取特定任务状态**
访问：`http://your-domain/api/tasks/{task_id}`

**取消任务**
POST请求：`http://your-domain/api/tasks/{task_id}/cancel`

#### 5. 异步搜索功能

**异步微博搜索**
```javascript
// POST /search
{
    "keyword": "搜索关键词",
    "max_page": 10,
    "async": "true"
}
```

**异步抖音搜索**
```javascript
// POST /search/douyin
{
    "keyword": "搜索关键词",
    "max_page": 5,
    "async": "true"
}
```

返回任务ID，可通过任务管理接口查询进度和结果。

#### 6. 爬虫统计信息
访问：`http://your-domain/api/crawler/stats`

返回爬虫性能统计，包括：
- 成功/失败请求数
- 成功率统计
- 网络请求性能指标

#### 7. 高级缓存管理（第三阶段新增）

**高级缓存统计**
访问：`http://your-domain/api/cache/advanced/stats`

返回多层缓存详细统计：
- L1/L2缓存命中率
- 缓存大小和使用情况
- 性能指标分析

**清空高级缓存**
POST请求：`http://your-domain/api/cache/advanced/clear`

#### 8. 智能监控告警（第三阶段新增）

**获取告警信息**
访问：`http://your-domain/api/monitoring/alerts?limit=50`

**告警统计信息**
访问：`http://your-domain/api/monitoring/alerts/stats`

**手动触发告警检查**
POST请求：`http://your-domain/api/monitoring/check`

#### 9. 数据预处理统计（第三阶段新增）
访问：`http://your-domain/api/preprocessing/stats`

返回智能预处理统计：
- 处理记录数量

#### 10. 前端优化统计（第三阶段新增）
访问：`http://your-domain/api/frontend/stats`

返回前端优化信息：

## 配置参数说明

### 缓存配置
- `CACHE_ENABLED`: 是否启用缓存（true/false）
- `CACHE_TTL`: 缓存过期时间（秒）
- `MAX_CACHE_SIZE`: 最大缓存条目数

### 处理配置
- `BATCH_SIZE`: 批处理大小
- `MEMORY_LIMIT_MB`: 内存使用限制（MB）

### 并发配置（第二阶段新增）
- `MAX_WORKERS`: 最大工作线程数
- `MAX_CONCURRENT_REQUESTS`: 最大并发请求数
- `ENABLE_CONCURRENT_CRAWLING`: 是否启用并发爬虫

### 网络配置（第二阶段新增）
- `REQUEST_TIMEOUT`: 请求超时时间（秒）
- `REQUEST_RETRY`: 请求重试次数
- `REQUEST_DELAY`: 请求间隔（秒）

### 数据处理配置（第二阶段新增）
- `ENABLE_DATA_PIPELINE`: 是否启用数据流水线
- `PIPELINE_PARALLEL`: 是否启用流水线并行处理

### 监控配置（第二阶段新增）
- `MAX_METRICS`: 最大监控指标数
- `CLEANUP_INTERVAL_HOURS`: 清理间隔（小时）
- `ENABLE_MONITORING`: 是否启用系统监控

### 高级缓存配置（第三阶段新增）
- `L1_CACHE_SIZE`: L1内存缓存大小
- `L1_CACHE_TTL`: L1缓存过期时间（秒）
- `L2_CACHE_SIZE_MB`: L2文件缓存大小（MB）
- `CACHE_DIR`: 缓存文件目录

### 前端优化配置（第三阶段新增）
- `ENABLE_COMPRESSION`: 是否启用资源压缩
- `STATIC_CACHE_MAX_AGE`: 静态资源缓存时间（秒）

### 告警配置（第三阶段新增）
- `EMAIL_ALERTS_ENABLED`: 是否启用邮件告警
- `SMTP_SERVER`: SMTP服务器地址
- `SMTP_PORT`: SMTP端口
- `SMTP_USER`: SMTP用户名
- `SMTP_PASSWORD`: SMTP密码
- `ALERT_RECIPIENTS`: 告警接收邮箱（逗号分隔）

### 数据预处理配置（第三阶段新增）
- `STOPWORDS_FILE`: 停用词文件路径
- `TEXT_CLEANING_ENABLED`: 是否启用文本清洗
- `DATA_VALIDATION_ENABLED`: 是否启用数据验证

### 性能监控配置
系统会自动监控以下指标：
- CPU使用率
- 内存使用率
- 磁盘使用率
- 网络I/O
- 应用响应时间
- 缓存命中率

## 最佳实践

### 1. 内存管理
- 系统会自动进行内存清理
- 每100次请求执行一次垃圾回收
- 内存使用超过80%时自动清理缓存

### 2. 缓存策略
- 频繁访问的数据会自动缓存
- 缓存满时使用LRU策略淘汰
- 可根据需要调整缓存大小和过期时间

### 3. 批处理优化
- 大量数据处理自动使用批处理
- 可通过`BATCH_SIZE`调整批次大小
- 系统会根据内存使用情况动态调整

### 4. 性能监控
- 定期查看性能报告
- 关注系统资源使用情况
- 根据优化建议调整配置

## 故障排除

### 1. 内存使用过高
- 检查`MEMORY_LIMIT_MB`设置
- 减少`MAX_CACHE_SIZE`
- 增加内存清理频率

### 2. 缓存命中率低
- 检查`CACHE_TTL`设置
- 增加`MAX_CACHE_SIZE`
- 优化缓存键策略

### 3. 性能监控异常
- 检查psutil库是否正确安装
- 确认系统权限设置
- 查看错误日志

## 注意事项

1. **功能兼容性**：所有优化都保持现有功能完全不变
2. **数据安全**：CSV文件存储方式保持不变
3. **渐进优化**：采用分阶段优化策略，降低风险
4. **监控告警**：建议定期检查性能指标
5. **配置调优**：根据实际使用情况调整配置参数
